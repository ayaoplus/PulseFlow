# Midday Summary Template

用途：每天 15:30 生成一次“日间总结”。

## 目标

基于当前时间点之前已经发生的工作，做一版有判断的中场总结：
- 截止目前做了什么
- 这些动作说明了什么
- 下午/晚上最该继续推进什么
- 哪些事应该砍掉、延后或降级

## 必读数据源

由调用方提供实际路径。默认优先使用：

1. 当前 `NOW.md`
2. 今日 AI work logs
3. 今日 usage / token 汇总
4. 如存在，可补充读取各 agent 的当日状态文件
5. 如存在，可补充读取当天的运行时 memory

## 完成判定规则

读取 `NOW.md` 时，完成项不要只盯 `DONE`：

- `DONE` 中的 `- [x]`，当然算已完成
- `FOCUS` / `TODAY` / `UP NEXT` 中凡是明确打了 `- [x]` 的，也一律视为已完成
- 这些还没被手动移动到 `DONE` 的条目，仍应计入“截至目前已推进”
- 但如果它们还留在活跃区块里，可以顺手指出这是面板卫生问题，不要误判成未完成
- 只有未勾选的 `- [ ]` 才算待推进 / 未完成
- `ITERATION NOTES` 只作为方向性备注与上下文，不算待推进任务清单

## 输出骨架

按下面结构输出，但具体措辞、顺序、重点允许灵活生成：

### ⏰ 日间总结 · YYYY-MM-DD 15:30

### 1. 截至目前已推进
- 只写真正有价值的推进，不要流水账
- 优先写人类主任务的推进，再写协调者 / AI agent 的支持动作

### 2. 判断与分析
- 对当前节奏做判断：今天是推进顺畅、偏散、卡住、还是跑偏
- 明确指出最值得继续押注的方向
- 如果出现偏题、低价值忙碌、信息噪音，直接点出来

### 3. 下半天建议
- 按 `P0 / P1 / P2` 给建议
- `P0` 只允许 1-2 项
- 允许建议“别做某事”

### 4. 风险 / 阻塞
- 只列真正会影响今晚结果的阻塞
- 没有就不硬写

### 5. 一句话总评
- 一句短评，像主管理人在中场给判断

## 风格要求

- 简洁、直接、有判断
- 不要机械复述 NOW.md
- 不要把所有任务都重新列一遍
- 允许适度锋利，别当客服

## 发送与归档

- 由调用方决定最终 delivery 目标
- 同时归档到调用方指定的 `cron-summaries/midday/YYYY-MM-DD.md`
- 归档内容与最终输出正文保持一致
