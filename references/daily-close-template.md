# Daily Close Template

用途：每天 00:05 生成一次“前一日总结”。

## 目标

读取前一日的日报归档，结合任务盘与 usage 数据，输出一版更偏复盘视角的日结：
- 昨天真正完成了什么
- 哪些任务还剩着
- token 消耗大概花在哪种工作上
- 今天最应该接住什么

## 优先数据源

由调用方提供实际路径。默认优先使用：

1. 前一日日报归档 `daily-reports/YYYY-MM-DD.md`
2. PulseFlow 历史月文件
3. 当前 `NOW.md`
4. 前一日 AI work logs
5. usage / token 汇总
6. 如存在，可补充读取前一日运行时 memory

## 缺失时的回退规则

如果前一日日报归档不存在：
- 明确说明“前一日日报归档缺失，以下为根据任务盘与运行记录补建的日结”
- 再基于 history / NOW / AI logs / usage 补建，不要卡死

## 输出骨架

### 🌙 前一日总结 · YYYY-MM-DD

### 1. 昨天完成了什么
- 归纳，不要原样照抄
- 优先写关键完成，不写低价值碎活

### 2. 还剩什么
- 把未完成但仍重要的任务抽出来
- 区分“继续推进”与“可以暂缓”

### 3. Token 与投入概况
- 给出 token 消耗概览
- 简要解释这些 token 大致花在哪些工作类型上
- 不需要伪精确到每一小项

### 4. 复盘判断
- 对昨天的工作质量给判断
- 可以指出：推进有效 / 节奏散 / 过度切换 / 产出不错但没收口 等等

### 5. 今天建议
- 给今天的 `P0 / P1 / P2`
- 重点是承上启下，而不是重写一份完整计划

## 风格要求

- 更像复盘，不像播报
- 要有判断，不要堆材料
- 允许自由发挥，但必须围绕模板骨架

## 发送与归档

- 由调用方决定最终 delivery 目标
- 同时归档到调用方指定的 `cron-summaries/daily-close/YYYY-MM-DD.md`
  - 这里的 `YYYY-MM-DD` 指被总结的那一天（即昨天）
